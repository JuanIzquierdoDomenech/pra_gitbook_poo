# Introducción

<figure><img src=".gitbook/assets/logos.png" alt=""><figcaption></figcaption></figure>

En la Práctica 1 trabajaremos de forma práctica los **conceptos y técnicas sobre Programación Orientada a Objetos (POO) y Programación Genérica (PG)** que hemos aprendido en el Tema 1 de teoría.&#x20;

Concretamente, desarrollaremos un **proyecto informático para gestionar el backend de una potencial aplicación de dibujo 2D**. Esto probablemente os recordará a la Práctica 5 de la asignatura "Programación". Pues bien, vamos a desarrollar un proyecto inspirado en dicha práctica, pero sacrificando algunas funcionalidades para poner el **foco en el diseño de interfaces, jerarquías de clases, y clases genéricas**, aplicando técnicas de programación avanzadas. La práctica se divide en dos partes: &#x20;

* [**Parte 1: Diseño de una interfaz genérica para una estructura de datos lineal de tipo lista**](parte-1-edl-lista-generica/), y la concreción de esta interfaz en **dos implementaciones concretas**: mediante representación de secuencias basadas en memoria contigua (arrays), y mediante memoria dispersa (nodos enlazados).&#x20;
* [**Parte 2: Diseño e implementación de las clases que permitan modelar las entidades de una aplicación de dibujo 2D**](parte-2-jerarquia-de-dibujo-2d/).&#x20;

Como pre-requisito, **deberéis haber realizado previamente la** [**Práctica 0**](https://app.gitbook.com/o/uY9F7U6WRW2jZBN5gWuV/s/SyyZos0HKvzFTjkdzLNv/), para familiarizaros con las herramientas del entorno de trabajo: sistema operativo Linux, terminal Bash, programación en C++, edición de ficheros de código fuente con Vim, control de versiones con Git, y automatización de procesos de compilación con Make.&#x20;

{% hint style="danger" %}
Recordad que **en los exámenes de prácticas se evaluará el uso de estas herramientas informáticas.**
